SETUP Before You Open Claude Code

How to use this document

→
Each numbered prompt is a separate Claude Code session. Start a new session for each major prompt. Don't chain everything into one massive session — Claude Code works best with focused, scoped tasks.
→
Copy the prompt exactly. Every word is deliberate. The tech stack, naming conventions, and constraints are specified to prevent Claude Code from making bad choices.
→
Review before moving to the next prompt. Each prompt builds on the previous. If something breaks or looks wrong, fix it before proceeding. Don't stack broken code.
→
Commit to GitHub after every prompt. Small commits. If a prompt breaks something you can always roll back. This is your safety net.
→
Deploy to Vercel after Prompt 2. Your site should be live at robotomated.com by end of Day 3. Then every subsequent build deploys automatically.

RULES Non-Negotiable Ground Rules

These protect the business long-term

1
Never pay to influence a RoboScore. The entire platform's value depends on editorial independence. If manufacturers offer money to improve scores, the answer is always no. Sponsored placement (clearly labeled) is fine. Score manipulation is not.
2
Commit to GitHub after every prompt. Small commits. Descriptive messages. If something breaks you need to be able to roll back fast.
3
Ship early, imperfect is fine. The homepage doesn't need to be perfect. The advisor doesn't need every feature. Get it live, get real users, let real feedback drive what you build next.
4
Publish before you're ready. The first 5 reviews don't need to be your best work. The SEO clock starts when the page is indexed — not when you think it's perfect.
5
Every API key goes in .env, never in code. Anthropic key, Supabase keys, Stripe keys. If you accidentally commit a key, rotate it immediately.
6
When Claude Code gets stuck, break it smaller. If a prompt produces broken code, don't try to fix it in the same session. Start fresh, scope it narrower, and rebuild that piece.
